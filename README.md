项目概述：

本项目为数据结构课程实践项目，以校园导航场景为载体，从基础信息管理到路径规划逐步迭代，共6个完整可运行版本。每个版本在前序功能基础上扩展新能力，对应不同的数据结构与算法知识点，完整覆盖线性表、栈、队列、字符串匹配、排序、图、哈希表等核心内容。

版本迭代总览：

版本	核心功能	核心数据结构	核心算法	

版本1	地点与分类增删改查、按分类筛选	结构体数组、线性表	顺序遍历查找	

版本2	浏览历史后退、最近5个访问地点	顺序栈、循环队列	入栈/出栈、入队/出队	

版本3	地点名称模糊搜索、双算法效率对比	字符数组	朴素模式匹配、KMP模式匹配	

版本4	距离/热度双向排序、热力等级展示	结构体数组	快速排序（多维度比较）	

版本5	全路径查询、最短路径推荐	邻接矩阵图	DFS深度优先搜索、Dijkstra最短路径	

版本6	哈希快速查找、排序结果直接跳转	链地址法哈希表	BKDR哈希、哈希查找、快速排序	

数据结构与算法清单：

数据结构

1. 线性表：数组存储地点、分类基础信息，支持增删改查

2. 栈：顺序栈实现浏览历史回溯，满足“后进先出”的后退逻辑

3. 队列：循环队列实现最近访问记录，固定容量5，超出自动淘汰最早记录

4. 图：邻接矩阵存储校园地点间的连通关系与路径距离

5. 哈希表：链地址法解决哈希冲突，构建地点名称索引，实现快速定位

算法：

1. 查找类：朴素遍历查找、KMP字符串模式匹配、哈希表查找

2. 排序类：快速排序，支持距离、热度多维度双向排序

3. 图算法：DFS深度优先遍历（全路径枚举）、Dijkstra单源最短路径

版本一：
维度	复杂度	说明	
时间复杂度	新增：O(1)
查找/修改/删除：O(n)
按分类筛选：O(n)	尾插新增为常数操作；其余操作均依赖顺序遍历数组	
空间复杂度	O(n + k)	仅用数组存储地点和分类数据，无额外辅助空间	

版本二：
维度	复杂度	说明	
时间复杂度	入栈/出栈：O(1)
入队/出队/去重：O(1)	栈和队列均为固定操作，队列容量固定为5，属于常数级规模	
空间复杂度	O(n)	主体数据存储仍为O(n)，栈和队列为线性小空间，不改变量级	

版本三：
维度	复杂度	说明	
时间复杂度	朴素匹配：
  最坏 O(n × L × m)
  平均 O(n × (L + m))

KMP匹配：
  最坏/平均 O(n × (L + m))	朴素匹配在极端失配下呈平方级；KMP通过next数组避免回溯，稳定线性	
空间复杂度	O(n + m)	需额外开辟O(m)的next数组，其余为原数据存储	

版本四：
维度	复杂度	说明	
时间复杂度	平均：O(n log n)
最坏：O(n²)	快速排序平均性能最优；极端有序场景退化为平方级，实际极少出现	
空间复杂度	O(n)	需复制一份数组用于排序；快速排序递归栈平均O(log n)，可忽略	

版本五：
维度	复杂度	说明	
时间复杂度	DFS全路径：
  最坏 O(n!)
  稀疏图：多项式级

Dijkstra最短路径：O(n²)	完全连通图中路径数量呈阶乘增长；邻接矩阵实现的Dijkstra稳定为平方级	
空间复杂度	O(n²)	邻接矩阵存储边权占用平方级空间；DFS递归栈、路径存储为O(n)	

版本六：
维度	复杂度	说明	
时间复杂度	哈希查找：
  平均 O(1)
  最坏 O(n)（全冲突）

排序：同版本4，平均O(n log n)	哈希查找平均为常数级；极端全冲突下退化为链表遍历	
空间复杂度	O(n)	哈希表节点与原数据一一对应，额外空间线性增长
